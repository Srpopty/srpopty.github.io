发表于 2018-10-27 分类于 CTF
Orange的一道代码审计题目，整个比赛期间的第一道web题，鄙人也认为是很难的一道题(1700多个队只解出来4个)，赛后看了PDKT的题解，发现有很多可以在代码审计中学习的地方，还用到了前段时间在BlackHat USA上新提出的PHP反序列化技术，这里将分析一波。

Baby Cake

Get the shell plz!!!!!

题目简单明了，直接给了全部源码 (不像某些国内垃圾比赛全靠脑洞做题)，整个网站实现的功能就是根据 url 参数去请求目标网页，如果是 Get 的话会存 cache 在本地，Post 的话将会直接把 data 参数 post 到目标网站，并且不会保存 cache。

这里的请求方式调用了 httpclient ($method, $url, $headers, $data) 函数，下面看看这个函数做了什么

在这个函数中才真正调用了 CakePHP 的远程请求 API$http->$method ($url, $data, $options)，跟进到 CakePHP 的 API 中

Client 类是从 Cake\Http\Client 中调用的，这个类位于 \vendor\cakephp\cakephp\src\Http\Client.php 中

post 函数

这两个函数最后进行请求的时候都调用了 $this->_doRequest (Request::METHOD_POST|Request::METHOD_GET, $url, $data, $options);，继续跟进

又调用了 $this->_createRequest ($method,$url,$data,$options); 继续跟 _createRequest 看它对输入的 url 和 data 都做了什么

第 20 行在请求的时候创建了一个 Request 对象进行请求 new Request ($url, $method, $headers, $data);
Request 类位于 \Cake\Http\Client\Request，在 \vendor\cakephp\cakephp\src\Http\Client\Request.php 中，继续跟进

在第 25 行的构造函数中使用 $this->body ($data) 将 data 放在了 body 中，而在 body 函数中有一行代码判断 data 是否为数组，body 函数的文档中还提到了 Array data will be serialized with Cake\Http\FormData, 也就是说如果是数组的话会 new 一个 FormData 对象，并且将 data 添加到 FormData 中

跟到 FormData 中，位于 Cake\Http\FormData

在 addMany ($body) 函数中将 data 数组进行了处理，将 data 数组里的每一个键和值都用 $this->add ($name, $value) 加入到 body 中

再看 add 函数

在 add 函数中第 7 行代码出了问题
elseif (is_string ($value) && strlen ($value) && $value [0] === '@')
如果 value 是 “@” 开头的字符串的话会转入 addFile 中

由于此时进来的 $value 是一个字符串，所以流程会转入 else 中，而在 else 里第 10 行代码可以直接读取文件
$content = file_get_contents ($value);
读取到的内容在 $content 变量中，而 $content 会存入 FormDataPart 里

最后再次将 part 加入 add ($part)，所以读取到的 content 会存在 body 中，这样就构成了一个本地文件读取漏洞，
